# Releases

Phial is still evolving quickly. Expect conventions and APIs to tighten as the framework stabilizes.

When preparing a release:

- review public API changes in `phial`
- update templates and examples
- update docs before publishing
- verify `phial dev`, `phial prepare`, `phial build`, and `phial start`

Breaking changes should be documented clearly, especially when they affect:

- directory conventions
- generated type files
- route ownership rules
- runtime configuration
