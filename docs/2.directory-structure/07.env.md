# Environment Files

Horn follows the normal Vite environment model.

Common files:

- `.env`
- `.env.development`
- `.env.production`

Rules of thumb:

- values prefixed with `VITE_` are safe to expose to the client bundle
- server code can read environment variables directly at request or build time
- keep secrets out of the client bundle

Use environment files for deployment configuration, not for route data that changes per request.
